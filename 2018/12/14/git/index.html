Git学习之路

fxj 发表于 2018-12-14

  • 分布式VCS：服务端和客户端都有完整的版本库，脱离服务端，客户端照样可以管理版本，比集中式VCS更能提高版本管理效率
  • 最小配置
        配置user信息. global对当前用户所有仓库有效，其余两个参数为local和system(这个基本不用)

    显示config的配置可用git config --list --global

  • 创建仓库
        创建git仓库：（1）把已有的项目代码纳入Git管理：$ cd 项目文件夹, $ git init; (2) $ git init your_project_name, $ cd your_project_name

        基本命令: git add your_file, git commit -m"what you have done", git status, git log,git rm filename
  • .git目录
         HEAD文件记录的是当前处于的分支，config记录的与本仓库相关的信息(包括本地用户配置信息)，refs存放的分支和tag的各种信息
  • commit、tree和blob的关系
        一个commit对应一棵树，树可能会嵌套一些树以及有一些blob。在git根据文件内容产生blob，文件内容相同就代表同一个东西，只存储一个blob，跟文件名没关系
  • 一些基本操作命令
        变更文件名或文件位置（已被管理）：git mv 文件 新位置或新名, 清空暂存区所有内容：git reset --hard, 切换分支:git checkout new_branch
        查看版本演变历史：git log,简单显示则加上参数:git log --online, 只显示最近的几次提交历史则加上参数:git log -n4 --oneline,查看所有分支log：git log --all --graph, –graph是为了方便查看演进历史添加的参数

        基于某个分支创建并切换到新分支：git checkout -b new_branch old_branch; 删除分支：git branch -d branch_name,如果警告则根据提示改用-D

        修改最新一次commit的message：git commit --amend,然后进行编辑更改;修改以前的commit的message:git rebase -i his_parent_branch, 然后在编辑器里根据情况把pick变为r，保存然后在下一个编辑框里修改message然后保存;把连续的多个commit整理成1个，使commit历史更为清晰：git rebase -i their_parent_branch,根据情况把pick改为s,保存在下一个编辑框里写上理由然后保存;不连续的commit在编辑器里调整pick后面分支的顺序以及把对应的pick变为s，保存后可能出现警告，根据提示运行git rebase --continue进行理由添加

        比较两个commit的差异：git diff commit1 commit2,例如git diff HEAD HEAD~1HEAD~1表示HEAD的父亲, git diff --cached,–cached表示暂存区，跟HEAD之间的差异，头指针HEAD指向某个commit；git diff默认比较的是工作区与暂存区的区别,git diff -- file_name比较工作区与暂存区某个文件名的区别;查看不同提交指定文件的差异：git diff commit1 commit2 -- filename, 或者用分支名(是一个指针，指向该分支最近的一次提交)

        变更暂存区用reset,变更工作区用checkout。使暂存区恢复成HEAD的状态,暂存区所做的变更都不要了：git reset HEAD;使工作区恢复成和暂存区一样:git checkout -- filename;取消暂存区部分文件的更改：git reset HEAD -- filename;

        消除最近几次提交：git reset --hard preserved_newest_branch,HEAD和暂存区还有工作区都会改变(小心使用)，例如git reset --hard HEAD,即取消工作区和暂存区所做的全部更改;

        把当前状态存起来，进行其他操作，做完之后再把存的给弹出来:git stash,git stash list, git stash pop或git stash apply,apply会保留,pop直接弹出

        .gitignore告诉git仓库哪些文件不需要纳入版本控制系统

        本地协议分为哑协议与智能协议，智能协议有进度条，传输速度也更快(打包压缩），更推荐使用。哑协议:git clone --bare file_path/.git another_path，智能协议:git clone --bare file://file_path/git another_path
